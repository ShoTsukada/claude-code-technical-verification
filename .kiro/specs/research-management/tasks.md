# Implementation Plan

## Task Format Notes
- `(P)` = 並行実行可能（他タスクと同時に作業できる）
- `- [ ]*` = 任意のテストカバレッジ（MVP後に対応可）

- [x] 1. データベーススキーマとデザインシステムの基盤を構築する

- [x] 1.1 Supabaseに全テーブルとインデックスを作成する
  - `institutions`（名称・所在地・設立年・概要）テーブルを作成し、名称カラムにインデックスを付与する
  - `research_themes`（タイトル・分野・説明・開始年）テーブルを作成し、分野カラムにインデックスを付与する
  - `investors`（氏名/組織名・種別・連絡先・投資分野）テーブルを作成し、`type IN ('individual', 'corporate')` CHECK制約を付与する
  - 3本の中間テーブル（`institution_themes`・`institution_investors`・`theme_investors`）を作成し、各ペアに UNIQUE 制約と ON DELETE CASCADE を設定する
  - `feedbacks`（カテゴリ・タイトル・内容・スコア・投稿元URL・画面名・ステータス・対応コメント）テーブルを作成し、カテゴリ・ステータス・スコアに CHECK制約を付与する
  - _Requirements: 1.1, 2.1, 3.1, 4.1, 4.5, 7.2, 7.6_

- [x] 1.2 DADSに準拠したデザイントークンをグローバルCSSに設定する
  - `app/globals.css` の `@theme` ブロックに行政ブルー（`#0017C1`）を起点とするカラートークンを定義する
  - テキスト・背景・ボーダー・セマンティック（成功/エラー/警告/情報）・フォーカスリングのカラートークンを追加する
  - 8px基準のスペーシングスケール、エレベーション（シャドウ）、角丸トークンを定義する
  - Noto Sans JP（weight: 400, 700）を `next/font/google` で読み込み、`--font-sans` 変数に割り当てる
  - _Requirements: 6.5_

- [x] 2. 共通レイアウトとUIコンポーネントを構築する

- [x] 2.1 グローバルレイアウトとサイドバーナビゲーションを実装する
  - `app/layout.tsx` にデスクトップ用左サイドバー（240px固定）＋メインコンテンツ（最大幅960px）の2カラムレイアウトを実装する
  - サイドバーに「研究機関」「研究テーマ」「投資家」「フィードバック」「フィードバック管理」へのリンクを配置し、現在のパスをアクティブ状態でハイライトする
  - 768px未満のモバイル画面ではサイドバーを非表示にし、ハンバーガーボタンでドロワー開閉できるようにする
  - サイドバーリンクにはLucide Reactのアイコンを併用し、絵文字は使用しない
  - _Requirements: 6.1, 6.5_

- [x] 2.2 (P) Toast通知コンポーネントを作成する
  - 成功・エラー・警告・情報の4種類に対応したトースト通知コンポーネントを作成する
  - DADS Notification Bannerの Color Chip スタイル（左ボーダー＋背景色）を適用し、カテゴリ別アイコンを付与する
  - `aria-live="polite"` を設定してスクリーンリーダーへの通知を確保する
  - 3〜5秒で自動消去し、手動で閉じるボタンも提供する
  - `app/layout.tsx` に配置して全画面から呼び出せる Context または状態管理フックと組み合わせる
  - _Requirements: 6.2_

- [x] 2.3 (P) ローディングインジケーターと確認ダイアログを作成する
  - ボタン内スピナー（`animate-spin`）とページ全体のスケルトンUI（`animate-pulse`）を作成する
  - 削除・解除操作用の確認ダイアログ（`aria-modal="true"`、Escape キーで閉じる、フォーカストラップ付き）を作成する
  - 確認ダイアログの「削除」ボタンは danger バリアント（赤）を使用し、「キャンセル」ボタンと視覚的に区別する
  - _Requirements: 1.6, 2.6, 3.6, 4.4, 6.3_

- [x] 4. 研究機関管理画面を実装する
